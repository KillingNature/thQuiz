# Gigaschool AI-Quiz Telegram Bot

Telegram-бот для прохождения AI-квиза, сбора лидов и контентных рассылок.

---

## Что делает бот

### Для пользователей
1. **AI-квиз** — 6 вопросов о том, как человек использует ИИ в работе
2. Сбор email после прохождения квиза
3. Отправка результатов (AI-профиль + 10 инструментов) в чат и на почту
4. Получение контентных рассылок (посты, кейсы, анонсы, вебинары)

### Для администраторов
- Создание и управление постами 4 типов
- Планирование рассылок по дате и времени (МСК)
- Предпросмотр постов перед отправкой
- Сегментация рассылок по меткам (тегам)
- Deep-link воронки для вебинаров (`?start=webinar_x`)
- Универсальные кнопки-ссылки в старте и постах
- Сбор заявок через расширенные формы
- Экспорт данных в CSV

## Типы постов (для маркетологов)

### 📝 Обычный пост (`/newpost`)
Стандартный контентный пост. Может содержать текст с форматированием, эмодзи и фото. Пользователь просто получает и читает его.

**Примеры использования:**
- Полезные материалы и советы по AI
- Статьи и разборы инструментов
- Новости и обновления

**Как создать:**
1. Напишите `/newpost`
2. Отправьте сообщение с текстом поста (можно прикрепить фото)
3. Бот сохранит пост — вы получите его ID

---

### 🧩 Интерактив-кейс (`/newcase`)
Пост с ситуацией и вариантами ответа. Пользователь выбирает вариант, после чего видит экспертный разбор. Повышает вовлечённость аудитории.

**Пример:**
> *«Клиент просит внедрить AI в CRM-систему. Бюджет ограничен. Что вы предложите?»*
> 
> Кнопки: `Чат-бот для поддержки` | `AI-скоринг лидов` | `Автоматизация отчётов`
>
> → После выбора: *экспертный разбор с правильным подходом*

**Как создать (3 шага):**
1. Напишите `/newcase`
2. **Шаг 1:** Отправьте описание ситуации (с фото, если нужно)
3. **Шаг 2:** Отправьте варианты ответов — **каждый с новой строки**:
   ```
   Добавить AI-чат в продукт
   Автоматизировать обработку заявок
   Внедрить рекомендательную систему
   ```
4. **Шаг 3:** Отправьте экспертный разбор — текст, который увидит пользователь после выбора

---

### 💰 Пост с формой (`/newsale`)
Продажный/анонсный пост, к которому автоматически добавляется кнопка **«Оставить заявку»**. При нажатии пользователь заполняет форму: имя, телефон, email, ник в Telegram. Данные сохраняются и доступны через `/export_leads`.

**Примеры использования:**
- Анонс курса с призывом к действию
- Специальное предложение
- Запись на консультацию

**Как создать:**
1. Напишите `/newsale`
2. Отправьте текст анонса (с фото, если нужно)
3. Кнопка «Оставить заявку» добавится автоматически

**Форма собирает:**
| Поле | Пример |
|------|--------|
| Имя | Иван Петров |
| Телефон | +7 900 123-45-67 |
| Email | ivan@mail.ru |
| Ник в Telegram | @ivanpetrov |

---

### 📢 Анонс вебинара (`/newwebinar`)
Пост с анонсом мероприятия, поддержкой меток и deep-link flow.

**Как создать (3 шага):**
1. Напишите `/newwebinar`
2. **Шаг 1:** Отправьте текст анонса (с фото, если нужно)
3. **Шаг 2:** Отправьте slug вебинара (например `webinar_27`) или `-`, если сегментация не нужна
4. **Шаг 3:** Отправьте ссылку на регистрацию (например `https://gigaschool.ru/webinar`)

Если задан `slug`, в посте будет кнопка **«Записаться»**, которая ставит метку **`webinar_27`** (как у диплинка). Дальше напоминания — через `/target ID webinar_27`.

## Стартовое сообщение

Это первое, что видит пользователь при нажатии `/start`. По умолчанию — стандартный текст с описанием квиза. Можно заменить на любой свой текст с фото.

**Как изменить:**
1. Напишите `/set_start`
2. Отправьте новое приветствие (текст, фото + подпись, эмодзи, форматирование)
3. Кнопка «Начать тест» добавится автоматически
4. `/preview_start` — проверить как выглядит
5. `/reset_start` — вернуть стандартный текст

### Дополнительные кнопки-ссылки в старте
- Одна кнопка: `/set_start_button Скачать гайд | https://disk.yandex.ru/...`
- Несколько кнопок: `/set_start_buttons` — затем одним сообщением построчно `Текст | URL`

Подходит для ссылок на Яндекс.Диск, лендинги, PDF и т.д.

## Webinar deep-link flow (для рекламы)

Для новой аудитории можно запустить отдельный старт по ссылке:

`https://t.me/ИМЯ_БОТА?start=webinar_27`

### Как настроить
1. Создайте flow: `/new_webinar_flow webinar_27`
2. Шаг 1/4: стартовое сообщение (текст/фото) — можно с вопросом и вариантами ответа **на следующем шаге**
3. Шаг 2/4: интерактивные кнопки под стартом: `Текст | https://...` для ссылок; **несколько строк простым текстом** — кнопки выбора (как кейс-пост); одна такая строка = кнопка «записаться» в боте; явная запись при смеси с ответами — строка с префиксом `+`, например `+Записаться`; `-` = только «✅ Записаться»
4. Шаг 3/4: сообщение после выбора варианта или после записи (текст / фото с подписью)
5. Шаг 4/4: опционально кнопка под этим сообщением: `Текст | URL` или `-`

После этого по `?start=webinar_27` пользователь сразу получает метку **`webinar_27`**. То же имя метки ставится при «Записаться» из анонса или из кнопки в flow. Отдельных меток за ответы нет. Рассылка участникам: `/target ID webinar_27`.

## Форматирование постов

Telegram поддерживает HTML-форматирование в сообщениях. Вы можете использовать:

- **Жирный текст** — просто выделите текст жирным в Telegram
- *Курсив* — выделите курсивом
- `Моноширинный` — выделите моноширинным
- ~~Зачёркнутый~~ — зачеркните текст
- [Ссылки](https://example.com) — вставьте ссылку
- 🎉 Эмодзи — используйте любые эмодзи

> **Важно:** пост будет выглядеть ровно так, как вы его отправите боту. Используйте `/preview ID`, чтобы проверить результат перед рассылкой.

### Как проверить пост перед отправкой

1. Создайте пост (например `/newpost`)
2. Отправьте содержимое
3. Используйте `/preview ID` — бот отправит вам пост так, как его увидят подписчики
4. Если всё ОК — `/schedule ID 2026-03-17 10:00` или `/send_now ID`
5. Если нужно переделать — `/delete_post ID` и создайте заново

## Настройка

### 1. Клонировать репозиторий
```bash
git clone https://github.com/KillingNature/thQuiz.git
cd thQuiz
```

### 2. Создать виртуальное окружение
```bash
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или venv\Scripts\activate  # Windows
```

### 3. Установить зависимости
```bash
pip install -r requirements.txt
```

### 4. Создать файл .env
```bash
nano .env
```

Содержимое:
```
BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
SMTP_EMAIL=quiz@gigaschool.ru
SMTP_PASSWORD=пароль_приложения_яндекс
ADMIN_IDS=123456789,987654321
```

#### Где взять значения

**BOT_TOKEN:**
1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. `/newbot` → выберите имя и username
3. Скопируйте токен

**SMTP_EMAIL и SMTP_PASSWORD (Яндекс):**
1. Зайдите в [Яндекс.Почту](https://mail.yandex.ru) → Настройки → Безопасность
2. Включите «Пароли приложений»
3. Создайте пароль для «Почта — IMAP/SMTP»
4. `SMTP_EMAIL` = ваш адрес, `SMTP_PASSWORD` = сгенерированный пароль

**ADMIN_IDS:**
1. Откройте [@userinfobot](https://t.me/userinfobot) в Telegram
2. Нажмите `/start` — бот покажет ваш ID
3. Можно указать несколько ID через запятую: `123456789,987654321`

### 5. Запустить
```bash
python bot.py
```

## Деплой на Yandex Cloud (Ubuntu)

### Подготовка сервера
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install python3 python3-venv python3-pip git -y
```

### Клонирование и настройка
```bash
cd ~
git clone https://github.com/KillingNature/thQuiz.git
cd thQuiz
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
nano .env   # заполнить BOT_TOKEN, SMTP_EMAIL, SMTP_PASSWORD, ADMIN_IDS
```

### Настройка systemd
```bash
sudo nano /etc/systemd/system/quizbot.service
```

Содержимое (замените `ВАШ_ЮЗЕР` на реального пользователя):
```ini
[Unit]
Description=Gigaschool AI-Quiz Telegram Bot
After=network.target

[Service]
Type=simple
User=ВАШ_ЮЗЕР
WorkingDirectory=/home/ВАШ_ЮЗЕР/thQuiz
ExecStart=/home/ВАШ_ЮЗЕР/thQuiz/venv/bin/python bot.py
Restart=always
RestartSec=5
EnvironmentFile=/home/ВАШ_ЮЗЕР/thQuiz/.env

[Install]
WantedBy=multi-user.target
```

Запуск:
```bash
sudo systemctl daemon-reload
sudo systemctl enable quizbot
sudo systemctl start quizbot
```

### Полезные команды
```bash
sudo systemctl status quizbot         # статус
sudo journalctl -u quizbot -n 50 -f   # логи в реальном времени
sudo systemctl restart quizbot         # перезапуск
sudo systemctl stop quizbot            # остановка
```

### Обновление бота (после изменений в коде)

## Рассылка по расписанию

Бот автоматически проверяет запланированные посты **каждые 5 минут**.

**Алгоритм:**
1. Создаёте пост (`/newpost`, `/newcase`, `/newsale`, `/newwebinar`)
2. Планируете дату и время: `/schedule ID 2026-03-17 10:00`
3. (Опционально) Ограничиваете пост сегментом: `/target ID webinar_27` (участники этого вебинара)
4. В назначенное время (МСК) бот отправляет пост всем или только выбранному сегменту
5. Пользователи, которые присоединились после отправки, **не** получат старые посты

**Время — московское (UTC+3).** Например, `/schedule 5 2026-03-20 09:00` отправит пост 20 марта в 9:00 по Москве.

### Таблицы в базе данных

| Таблица | Описание |
