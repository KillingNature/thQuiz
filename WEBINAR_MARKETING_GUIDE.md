# Быстрый сценарий: запуск вебинара в боте

Это короткая инструкция для маркетолога: что нажимать, зачем это нужно и что должно получиться.

---

## Шаг 0. Подготовка

**Что сделать**
- выбрать slug вебинара (например `webinar_1`)
- подготовить ссылку на лендинг/регистрацию
- подготовить тексты: анонс, подтверждение, напоминания

**Зачем**
- slug нужен для сегментации и deep-link

**Что получится**
- единый идентификатор вебинара: `webinar_1`

---

## Шаг 1. Настроить стартовый вебинарный flow (для новой аудитории)

**Команда**
- `/new_webinar_flow webinar_1`

**Что здесь происходит**
- бот попросит **4 шага**:
  1) стартовое сообщение (текст/фото)  
  2) текст подтверждения после «Записаться»  
  3) **интерактивные кнопки** под стартом (несколько ссылок + одна кнопка записи)  
  4) опционально — ещё одна ссылка под текстом подтверждения  

**Формат шага 3 (кнопки):** каждая строка — одна кнопка.  
- ссылка: `Текст | https://...`  
- запись: строка **без** ` | `, например `Записаться на вебинар`  
- если отправить `-` — только кнопка «✅ Записаться» без лишних ссылок  

**Зачем**
- чтобы люди из рекламы по ссылке `?start=webinar_1` видели вебинарный сценарий с нормальным интерактивом (как у постов)

**Что получится**
- под стартом — ваши кнопки (лендинг, материалы и т.д.) + запись на вебинар  
- при записи ставится метка `webinar_1_optin`

---

## Шаг 2. Поставить правильную ссылку в лендинг/рекламу

**Ссылка**
- `https://t.me/gigaschool_aiproduct_bot?start=webinar_1`

**Зачем**
- это вход новой аудитории сразу в вебинарный flow

**Что получится**
- пользователь попадет в бот, увидит вебинарный экран и сможет записаться

---

## Шаг 3. Отправить анонс текущей базе бота

**Команда**
- `/newwebinar`

**Что здесь происходит**
- бот попросит:
  1) текст/фото анонса  
  2) slug (`webinar_1`)  
  3) ссылку на регистрацию

**Зачем**
- собрать в тот же сегмент старую аудиторию, которая уже есть в боте

**Что получится**
- пост с кнопкой "Записаться", которая тоже ставит метку `webinar_1_optin`

---

## Шаг 4. Отправить анонс

**Команды**
- сразу: `/send_now ID`
- по времени: `/schedule ID 2026-03-17 10:00`

**Зачем**
- чтобы запустить набор людей на вебинар

**Что получится**
- анонс уйдет аудитории

---

## Шаг 5. Проверить, что люди записываются

**Команда**
- `/tags`

**Что здесь происходит**
- бот покажет список меток и количество людей в каждой

**Зачем**
- увидеть размер сегмента вебинара

**Что должно быть**
- в списке есть `webinar_1_optin: N` (N растет)

---

## Шаг 6. Делать напоминания только записавшимся

**Команды**
1) создать напоминание: `/newpost`  
2) поставить сегмент: `/target ID webinar_1_optin`  
3) отправить: `/send_now ID` или `/schedule ID YYYY-MM-DD HH:MM`

**Зачем**
- не спамить тем, кто не записался

**Что получится**
- напоминание получат только участники сегмента `webinar_1_optin`

---

## Шаг 7. Опционально: добавить кнопку-ссылку в пост

**Команда**
- `/set_button ID Текст | URL`

**Пример**
- `/set_button 25 Материалы вебинара | https://disk.yandex.ru/...`

**Зачем**
- дать быстрый доступ к лендингу, материалам, записи

**Что получится**
- в выбранном посте появится дополнительная кнопка-ссылка

---

## Шаг 8. Замерить результат рекламы

**Команды**
- до запуска: `/snapshot до_рекламы`
- после: `/compare`

**Зачем**
- понять рост базы и эффект кампании

**Что получится**
- сравнение "было/стало" по ключевым цифрам

---

## Мини-шпаргалка (в одну строку)

1. `/new_webinar_flow webinar_1`  
2. ссылка в лендинг: `https://t.me/gigaschool_aiproduct_bot?start=webinar_1`  
3. `/newwebinar` (с `webinar_1`)  
4. `/send_now ID`  
5. `/tags`  
6. `/newpost` -> `/target ID webinar_1_optin` -> `/schedule ...`

---

## Частые ошибки

- **Deep-link ведет в обычный старт**  
  Не настроен flow: повторите `/new_webinar_flow webinar_1`.

- **Напоминание ушло всем**  
  Перед отправкой не поставлен `/target ID webinar_1_optin`.

- **Кнопка-ссылка не сохраняется**  
  Формат только такой: `Текст | URL`, URL должен начинаться с `http`.
